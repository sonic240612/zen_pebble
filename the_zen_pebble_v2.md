# 🗿 [기획서 v2] The Zen Pebble : 극한의 정적 챌린지

## 1. 서비스 개요
*   **서비스 명:** The Zen Pebble (더 젠 페블)
*   **한 줄 정의:** 마우스와 키보드에서 손을 떼고 가장 오래 버티는 사람이 승리하는 '디지털 인내심 테스트'.
*   **핵심 가치:** #역설적재미 #디지털디톡스 #극한의긴장감 #단순함의미학
*   **타겟 유저:** 
    *   심심해서 아무거나 클릭하고 싶은 사람.
    *   자신의 인내심을 시험해보고 싶은 경쟁심 강한 사람.
    *   복잡한 세상에서 잠시나마 강제로 '정지'하고 싶은 사람.

## 2. 핵심 게임 메커니즘 (Core Loop)
**[접속] → [시작 버튼 클릭] → [완벽한 정적 유지] → [실시간 환경 변화 보상] → [움직임 발생 → 실패] → [랭킹 등록]**

*   **승리/패배 조건:**
    *   **시작:** 화면 중앙의 '시작' 버튼을 누르는 순간 타이머 작동.
    *   **실패(Death):** 다음과 같은 이벤트 발생 시 즉시 종료.
        *   마우스 커서의 이동 (1px이라도 움직임 감지)
        *   키보드의 아무 키나 입력
        *   마우스 클릭/드래그
        *   (모바일의 경우) 화면 터치 및 자이로 센서의 큰 흔들림 감지
    *   **보상:** 버틴 시간에 비례하여 **조약돌은 그대로 둔 채 주변 환경이 변화**함.

## 3. 상세 기능 설계

### ① 환경 진화 시스템 (Environmental Evolution System)
**핵심 컨셉:** 조약돌 자체는 **절대 변하지 않는 고정된 기준점**으로 남고, 시간의 흐름에 따라 **주변 세계만 변화**함. 이는 '나(조약돌)는 움직이지 않는데 세상만 변한다'는 선(禪)의 역설을 시각화.

| 단계 | 버틴 시간 | 환경 변화 (Surroundings) | 사운드 (Ambient) | 비고 |
| :--- | :--- | :--- | :--- | :--- |
| **1단계: 새벽** | 0~5분 | 정적인 흰색/회색 배경, 미세한 먼지 입자만 부유 | 낮은 화이트 노이즈 | 시작 상태 |
| **2단계: 아침** | 5~30분 | 배경에 옅은 안개 발생, 멀리서 산 실루엣 등장 | 새소리(지저귐) 추가 | 안개 밀도 시간에 따라 변화 |
| **3단계: 낮** | 30분~2시간 | 안개 걷히며 풍경 선명해짐 (숲/들판/호수 중 랜덤) | 시냇물/바람 소리, 곤충 울음소리 | **풍경 타입 3종 랜덤** |
| **4단계: 오후** | 2~6시간 | 그림자 방향/길이 변화, 구름 이동, 빛 각도 변화 | 바람 소리 강화, 나뭇잎 흔들림 | **실시간 태양 궤적 시뮬레이션** |
| **5단계: 황혼** | 6~12시간 | 노을 그라데이션, 긴 그림자, 반딧불이 등장 | 귀뚜라미/풀벌레 소리, 먼 종소리 | **노을 색상 5종 바리에이션** |
| **6단계: 밤** | 12~24시간 | 별자리 등장, 달 위상 변화, 은은한 달빛 | 깊은 정적, 간헐적 올빼미/개구리 소리 | **별자리 패턴 12종 (월별)** |
| **7단계: 깊은 밤** | 24~48시간 | 오로라/유성우 발생, 생물발광 식물 빛남 | 신비로운 앰비언트 드론 사운드 | **오로라 패턴 4종** |
| **8단계: 새벽 복귀** | 48시간~ | 서서히 밝아지며 1단계로 순환 (무한 루프) | 새벽 합창(Dawn Chorus) | **순환 시 '윤회 카운트' 증가** |

#### 🌿 바리에이션 시스템 (무한 재플레이성 확보)
같은 시간대라도 매 세션마다 다른 경험 제공:

| 변수 | 바리에이션 수 | 설명 |
| :--- | :---: | :--- |
| **기본 풍경 테마** | 4종 | 소나무숲 / 대나무숲 / 호수변 / 사막 모래언덕 |
| **계절** | 4종 | 봄(꽃잎) / 여름(무성) / 가을(단풍) / 겨울(눈) |
| **날씨** | 5종 | 맑음 / 구름조금 / 안개 / 비 / 눈 |
| **동물 방문자** | 12종+ | 시간대별 확률적 출현 (새, 다람쥐, 사슴, 나비, 개구리 등) |
| **천문 현상** | 8종 | 유성우, 일식/월식, 행성 정렬, 혜성, 오로라, 무지개, 달무리, 성운 |
| **조약돌 주변 미세 변화** | - | 이슬 맺힘, 서리, 눈 쌓임, 낙엽 쌓임, 꽃잎 떨어짐 (조약돌 자체 불변) |

**총 조합 수:** 4 × 4 × 5 × 12 × 8 = **7,680+ 고유 조합** → 몇 시간을 넘어 며칠을 버텨도 새로운 장면 연출 가능

### ② 조약돌: 불변의 중심 (The Immutable Pebble)
*   **시각:** 처음부터 끝까지 동일한 회색 조약돌 (크기/형상/텍스처 고정)
*   **역할:** 화면 정중앙 고정, 모든 환경 변화의 기준점
*   **미세 인터랙션 (조약돌 자체 변화 아님):**
    *   시간에 따른 조명 변화에 의한 그림자/하이라이트만 반응
    *   비 올 때 물방울 튐, 눈 올 때 눈송이 닿음 → **물리적 반응만 존재**
    *   동물 방문 시 조약돌 위에 잠시 앉았다 감 (조약돌은 움직이지 않음)

### ③ 글로벌 랭킹 시스템 (The Hall of Silence)
*   **실시간 리더보드:** 현재 전 세계에서 가장 오래 버틴 TOP 100 기록 표시.
*   **단계별 명예의 전당:** 각 환경 단계(아침/낮/오후/황혼/밤/깊은밤/새벽) 최초 도달자 기록 영구 보존.
*   **윤회 랭킹:** 48시간 순환(1윤회) 완료 횟수 기준 별도 랭킹.
*   **공유 기능:** 실패 시 **"나는 [00일 00시간 00분] 동안 정적을 유지하며 [단계: 예: 황혼의 호수]를 목격했습니다. 당신은 얼마나 버틸 수 있나요?"** 문구 + 현재 환경 스크린샷 자동 생성하여 공유.

### ④ 통계/기록 시스템 (Personal Archive)
*   **나의 여정:** 역대 최고 기록, 총 누적 명상 시간, 도달한 최장 단계, 윤회 횟수.
*   **타임라인:** 날짜별 버틴 시간 그래프, 어떤 풍경/날씨/동물을 만났는지 기록.
*   **도감 (Compendium):** 
    *   만난 동물/식물/천문현상 수집 (포켓몬 도감 스타일)
    *   "첫 만남" 시간 기록 → 다시 보기 기능

## 4. UX/UI 디자인 전략
*   **Concept:** **Extreme Minimalism + Deep Immersion** (극도의 단순함 속 깊은 몰입)
*   **UI 구성:**
    *   **중앙 (고정):** 조약돌 - 화면 크기에 관계없이 중앙 고정, 절대 움직이지 않음
    *   **상단:** 초박형 타이머 (일:시:분:초) - 폰트: 얇은 산세리프, 색상: 환경 대비되는 밝기 자동 조정
    *   **하단:** 현재 단계명 + 환경 힌트 (예: "황혼 · 호수 · 가을비 · 반딧불이") - 아주 작게
    *   **우측 하단:** 랭킹/도감/설정 아이콘 (호버/탭 시에만 나타남)
*   **전체 화면 활용:** 조약돌 외 모든 영역 = 환경 캔버스 (레터박스 없음, 풀스크린)
*   **인터랙션:** 
    *   시작: 버튼 클릭 → 페이드 인하며 환경 렌더링 시작
    *   **실패 (v2):** 화면 흔들림(Shake) + **조약돌 주변 환경이 급격히 붕괴/흩어짐** (조약돌만 고독하게 남음) → 역설적 상실감
        *   안개가 걷히듯 환경 요소들이 순서대로 페이드아웃
        *   마지막까지 남은 조약돌이 선명해졌다가 전체 화면 페이드 투 블랙
        *   결과 UI 등장
    *   성공(단계 전환): 부드러운 크로스 디졸브 + 환경 사운드 레이어 추가

## 5. 기술 구현 방안 (Technical Stack)

### 🛠 Frontend (Performance Critical - 48시간+ 연속 구동)
*   **Framework:** `React` + `TypeScript`
*   **Rendering:** `Canvas API` (WebGL via `Three.js` 또는 `PixiJS`) - DOM 조작 최소화, 60fps 유지 필수
*   **State Management:** `Zustand` 또는 `Jotai` (타이머/환경 상태 원자적 관리)
*   **Event Listeners (Passive mode로 성능 최적화):**
    ```js
    window.addEventListener('mousemove', onFail, { passive: true });
    window.addEventListener('keydown', onFail, { passive: true });
    window.addEventListener('touchstart', onFail, { passive: true });
    ```
*   **Animation Loop:** `requestAnimationFrame` 기반 고정 타임스텝 루프
*   **시간 동기화:** `performance.now()` + `Date.now()` 하이브리드 (탭 비활성화 시에도 정확한 시간 계산)
*   **에셋 관리:** 환경 텍스처/사운드 **지연 로딩(Progressive Loading)** - 단계 진입 시점에만 로드

### 🛠 백엔드 (Simple & Scalable)
*   **Database:** `Supabase` (PostgreSQL + Realtime 구독)
*   **Schema 설계:**
    ```sql
    -- scores 테이블
    id, nickname, duration_ms, stage_reached, cycles_completed, 
    environment_seed, encountered_entities[], created_at
    
    -- leaderboards 뷰 (Materialized View, 1분마다 갱신)
    -- compendium: 유저별 수집 현황
    ```
*   **API:** 
    *   `POST /api/score` - 기록 저장 (idempotent key로 중복 방지)
    *   `GET /api/leaderboard?type=overall|stage|cycle&limit=100`
    *   `GET /api/compendium/:userId`
    *   `GET /api/daily-seed` - 일일 공통 시드 (경쟁 공정성용 옵션)

### 🛠 환경 생성 시스템 (Procedural Generation)
*   **시드 기반 결정론적 생성:** `seed = hash(userId + startTimestamp)` → 같은 시드 = 같은 환경 시퀀스 (재현/검증 가능)
*   **레이어드 렌더링:**
    1. 배경 하늘/그라데이션 (셰이더)
    2. 원경 지형/산/구름 (패럴랙스)
    3. 중경 식생/물체 (인스턴싱)
    4. 근경 파티클/이슬/눈/낙엽 (GPU 파티클)
    5. 조약돌 (고정 메쉬 + 동적 라이팅)
    6. 동물/방문자 (스프라이트/스켈레탈 애니메이션)
    7. UI 오버레이
*   **천문 계산:** `Astronomy.js` 또는 자체 구현으로 태양/달 위치, 별자리, 위상 실시간 계산

## 7. 핵심 차별화 포인트 (Key Differentiators)

1. **조약돌의 불변성** - "변하지 않는 것"을 중심으로 "변하는 세상"을 관조하는 경험
2. **시간 스케일의 확장** - 분/시간 단위를 넘어 **일/주 단위**로 설계된 최초의 방치형 챌린지
3. **7,680+ 환경 조합** - 같은 시간을 버텨도 매번 다른 풍경/만남 제공
4. **수집/도감 요소** - 단순 인내심 테스트를 넘어 '관찰자'로서의 기록 욕구 자극
5. **윤회 시스템** - 48시간 순환 구조로 '끝'이 없는 무한 플레이 가능
6. **기술적 신뢰성** - 48시간+ 연속 구동 가능한 안정적인 타이머/렌더링 아키텍처
7. **실패 연출의 역설** - 조약돌은 그대로, 환경만 사라지는 붕괴 연출로 '잃음'의 미학 구현
